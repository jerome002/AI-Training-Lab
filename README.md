# AI Training & Data Curation Lab

This repository serves as a professional portfolio for my work in **AI Alignment**, **Supervised Fine-Tuning (SFT)**, and **Data Labeling**. As a Full-Stack MERN Developer, I use this space to bridge the gap between software engineering and high-integrity AI training.

## Project Overview: Developer Feedback Sentiment Analysis
The primary project in this lab involves the curation and annotation of a technical dataset focused on developer sentiments and bug reporting.

### Tools & Frameworks Used
- **Labeling Tool:** Label Studio
- **Framework:** HHH (Helpful, Honest, Harmless)
- **Data Formats:** CSV (Source), JSON (Exported Annotations)
- **Technical Focus:** MERN Stack (MongoDB, Express, React, Node.js)

## Skills Demonstrated
- **Data Curation:** Transforming raw developer feedback into structured JSON data.
- **Sentiment Classification:** Identifying Positive, Negative, and Neutral tones in technical discourse.
- **Bug Identification:** Isolating specific technical vulnerabilities and 500-level errors from general feedback.
- **Model Alignment:** Applying RLHF (Reinforcement Learning from Human Feedback) principles to ensure data accuracy.

##  Repository Structure
- `/data_samples`: Contains the exported `sentiment_analysis_v1.json` file.
- `README.md`: Documentation of methodologies and skills.

## Professional Application
This work is designed to improve LLM (Large Language Model) performance in technical support and coding assistance roles. By providing high-quality, "Gold Standard" labels, I help models better understand the nuances of software development troubleshooting.
